## 报警阈值

### 需要立即干预的情况：
1. **Pod崩溃**：任何Pod连续重启超过3次
2. **服务不可用**：Lobe Chat或Casdoor持续5分钟无法访问
3. **数据库连接失败**：应用无法连接数据库
4. **数据不一致**：用户数量或消息数量异常减少

### 需要关注的情况：
1. **资源使用率高**：内存使用持续超过80%
2. **响应时间慢**：页面加载时间超过5秒
3. **认证错误**：用户登录失败率超过10%

## 监控工具建议

### 命令行监控：
```bash
# 实时监控Pod状态
kubectl get pods -n lobe-chat -w

# 查看实时日志
kubectl logs -f deployment/lobe -n lobe-chat

# 资源监控
kubectl top pods -n lobe-chat --containers
```

### 日志关键字监控：
- **错误级别**：`ERROR`, `FATAL`, `panic`
- **认证问题**：`auth`, `login`, `token`, `session`
- **数据库问题**：`database`, `connection`, `migration`
- **应用启动**：`Ready in`, `Started successfully`

## 问题处理流程

### 1. 发现问题
1. 记录发现问题的时间、现象
2. 收集相关日志：`kubectl logs deployment/lobe -n lobe-chat --since=5m`

### 2. 初步诊断
1. 检查Pod状态：`kubectl describe pod <pod-name> -n lobe-chat`
2. 检查事件：`kubectl get events -n lobe-chat --sort-by='.lastTimestamp'`
3. 检查资源：`kubectl describe nodes`

### 3. 问题分类
- **P0 严重故障**：服务完全不可用 → 考虑回滚
- **P1 功能故障**：部分功能异常 → 尝试修复
- **P2 性能问题**：响应慢但可用 → 优化调整
- **P3 警告信息**：日志警告但不影响功能 → 监控观察

### 4. 执行处理
根据问题分类执行相应操作，参考《紧急回滚计划.md》
